ChatGPT Efficiency Test

For this project, I evaluated ChatGPT's efficiency and accuracy across three different domains: History, Social Science, and Computer Security. I set up the evaluation using a combination of tools and technologies that made the process both streamlined and effective:

  • MongoDB: I used MongoDB to store the dataset, which included questions, anticipated answers, and ChatGPT's responses.

  • Node.js: I developed a server-side application with Node.js to connect the database with the ChatGPT API and handle communication seamlessly.

  • ChatGPT API: The API allowed me to programmatically send questions to ChatGPT and retrieve its responses for analysis. This setup ensured efficient data handling, smooth API integration, and a clear process for evaluating ChatGPTâ€™s performance.

Dataset Overview

I organized a dataset of questions spanning three domains:

  1. History: Questions explore historical events, important figures, and cultural developments.

  2. Social Science: These questions focus on topics like sociological theories, human behavior, and cultural phenomena.

  3. Computer Security: This category includes technical questions on cybersecurity concepts, best practices, and threats.



Each question in the dataset includes:

  • The question text itself

  • An anticipated answer, which I determined through research

  • A placeholder for ChatGPT's responses, which was populared during testing.

The purpose of this evaluation is to assess how well ChatGPT aligns its answers with the anticipated ones and to measure its response times across these domains. This dataset represents a comprehensive testing ground to evaluate ChatGPT's capabilities.
